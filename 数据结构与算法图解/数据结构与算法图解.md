操作的速度，并不按时间计算，而是按步数计算

| 数据类型 | 定义                                                         |      |
| -------- | ------------------------------------------------------------ | ---- |
| 数组     | 一个含有数据的列表                                           |      |
| 集合     | 一种不允许元素重复的数据结构                                 |      |
| 链表     | 每个结点除了保存数据，它还保存着链表里的下一结点的内存地址   |      |
| 双向链表 | 它每个结点都含有两个链——一个指向下一结点，另一个指向前一结点。此外，它还能直接访问第一个和最后一个结点。 |      |

冒泡排序：比较交换相邻，使最大位于最右侧，每次使分区减少1			 *O*(*N*^2^)

选择排序：找到最小，与分区开头交换，每次使分区减1							*O*(*N*^2^)

> tip:二者的速度为常数比，下者更快

插入排序：从索引1开始，将索引对因元素抽出，对其左侧进行平移操作	*O*(*N*^2^)

重复值检查：哈希法*O*(*N*)/嵌套循环*O*(*N*^2^)

散列表（键值对，e,g,Python中的字典）

散列函数：将字符串转化为数字列

分离链接：字符串对因的数字列可能相同，将其关联到对应散列表索引，采取新的检索方法

栈：只能在末尾插入数据（压栈）；只能读取末尾的数据； 只能移除末尾的数据（出栈）。

使用实例：语法检查，如对括号进行匹配

队列：只能在末尾插入数据；只能读取开头的数据；只能移除开头的数据。

递归：调用自身与基准情况

计算机对递归的理解：调用栈，先调用的进行压栈，最后进入的进行出栈，无限递归会导致栈溢出

快速排列：任取轴，进行分区，对分区结果再进行取轴分区 *O*(*N logN*)

二叉树：每个结点的子结点数量可为 0、1、2。 如果有两个子结点，则其中一个子结点的值必须小于父结点，另一个子结点的值必须大于父结点。即大在右，小在左

**删除操作遵循以下规则： **

* 如果要删除的结点没有子结点，那直接删掉它就好。

* 如果要删除的结点有一个子结点，那删掉它之后，还要将子结点填到被删除结点的位置上。

* 如果要删除的结点有两个子结点，则将该结点替换成其后继结点。一个结点的后继结点，就是所有比被删除结点大的子结点中，最小的那个（即节点右侧的最左端）

* 如果后继结点带有右子结点，则在后继结点填补被删除结点以后，用此右子结点替代后继结点的父节点的左子结点。

图的广度优先搜索：设定初始当前顶点，访问每个顶点的邻接点；让顶点出队，将其设为当前顶点； 访问每个顶点的邻接点。*O*(*V + E*)

Dijkstra 算法：

(1) 以起步的顶点为当前顶点。

(2) 检查当前顶点的所有邻接点，计算起点到所有已知顶点的权重，并记录下来。

(3) 从未访问过（未曾作为当前顶点）的邻接点中，选取一个起点能到达的总权重最小的顶点，作为下一个当前顶点。

(4) 重复前 3 步，直至图中所有顶点都被访问过

用大 *O* 来描述一个算法需要多少空间：当所处理的数据有 *N* 个元素时，该算法还需额外消耗多少元素大小的内存空间

将不消耗额外的内存空间的空间复杂度描述为 *O*(1)
